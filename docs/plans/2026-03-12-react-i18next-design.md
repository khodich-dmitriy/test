# Дизайн i18n на react-i18next

## Цель

Добавить полноценную локализацию всего приложения с поддержкой русского, английского и китайского языков, запоминанием выбранного языка и переключением языка в header.

## Почему библиотека

Проект будет масштабироваться, поэтому локализация должна опираться на стандартный и расширяемый инструмент. Для этого используется `react-i18next` c `i18next`, а не самописный словарь.

## Архитектура

- Общая конфигурация i18n хранится в `src/shared/i18n`.
- Выбранный язык сохраняется в cookie, чтобы серверный layout и клиентский UI стартовали в одном и том же языке.
- Корневой `app/layout.tsx` читает cookie языка, выставляет `html lang` и оборачивает приложение в i18n provider.
- Header получает language switcher рядом с theme switcher.
- Все пользовательские строки выносятся в translation resources по namespaces.

## Языки

- `ru`
- `en`
- `zh`

## Поведение

- По умолчанию используется русский язык.
- При изменении языка в header:
  - вызывается `i18n.changeLanguage`;
  - обновляется cookie;
  - обновляется `document.documentElement.lang`;
  - выбор сохраняется между перезагрузками.

## Форматирование

- Дата и время форматируются по текущему языку.
- Пользовательские статусы, подписи кнопок, aria-label и тексты ошибок локализуются через словари.

## Footer

- Footer остается в визуальном стиле карточки.
- Добавляется мягкая верхняя тень/градиент, чтобы контент визуально уходил под footer плавнее.

- Тесты на рендер ключевых экранов в локализованном виде.
